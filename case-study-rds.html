Executive Summary

Rapid7's multi-product platform struggled with fragmented UI patterns, divergent codebases, and inconsistent accessibility standards. I led the architectural direction, strategy, and rollout of the Rapid7 Design System (RDS) and MUI migration, partnering across design, engineering, product, and accessibility teams.

Together, we delivered a unified, scalable UI framework now powering all major products across the portfolio—improving efficiency, consistency, accessibility, and design maturity at scale.

The Challenge

Rapid7's product suite evolved through acquisition and parallel development, resulting in systemic fragmentation that hurt both user experience and engineering velocity:

  • 17+ button variants across products
  • 6 different DataGrid implementations
  • Inconsistent spacing, typography, and theming
  • Little or no code–design parity
  • No unified component specification or governance
  • Accessibility audits failing across multiple product lines
  • Multiple duplicate component libraries creating engineering inefficiencies
  • Divergent UI patterns for core interactions

My Role

I defined the system vision and architecture, built the governance and adoption model, and partnered closely with engineering to migrate to an enterprise-grade MUI implementation.

Key Responsibilities

  • Design system strategy and vision
  • Information architecture and component taxonomy
  • Governance model and design ops processes
  • Measurement frameworks and KPIs
  • Stakeholder alignment and communication
  • Foundations: theming, tokens, spacing, typography

Collaborative Deliverables

  • 60+ fully accessible MUI components
  • Platform templates and interaction patterns
  • Comprehensive documentation and tooling
  • Code library with engineering team
  • Adoption rollout plans per product team

Constraints

The project needed to balance multiple technical and organizational challenges:

  • Six product teams with different tech stacks
  • Parallel product delivery could not be disrupted
  • Required full WCAG 2.2 AA compliance
  • Needed to support light/dark theming
  • Must unify visual language across acquisitions
  • Required measurable ROI to secure future investment

Research & Discovery

I executed a comprehensive foundational audit across all six products, analyzing components, patterns, and user workflows.

Component Inventory

  • Buttons, form fields, and navigation patterns across all states
  • Tables, filters, cards, metrics, and summary views
  • Data visualization interactions

Key Findings

  • High divergence in structure and behavior across products
  • Engineering teams re-implementing similar components
  • Designers spending excessive time manually aligning patterns
  • No shared documentation or specification model
  • Accessibility issues caused by inconsistent semantic structure

Business & User Insights

  • Fragmentation raised onboarding time for new users
  • Inconsistency made insights harder to interpret
  • Teams lacked trust in shared UI patterns
  • Engineers lacked clarity on what "done" looked like

Strategy & Architecture

I architected RDS around three scalable layers that both design and engineering could rely on:

Layer 1 — Foundations

  • Design tokens (color, spacing, typography, radii, motion)
  • Light/dark theming system
  • Grid and layout rules
  • Accessibility standards

Layer 2 — Components

  • 60+ fully documented UI components
  • MUI-based implementations with Rapid7-specific overrides
  • Component spec sheets (behavior, states, accessibility, API)

Layer 3 — Templates & Standards

  • Dashboard template
  • Listing template
  • Details template
  • Form template
  • UX Standards Pillars: Accessibility, Responsiveness, Usability, Governance

Design & Implementation

Component Design

  • Created spec-driven Figma components
  • Defined semantic structure and interaction states
  • Introduced unified motion principles
  • Standardized keyboard navigation patterns
  • Designed flexible variants supporting edge cases

Engineering Migration (MUI)

  • Worked with engineering to define theme overrides
  • Created design → code mapping model via Supernova
  • Redefined component APIs to align with RDS usage
  • Established release and versioning processes

Documentation

  • Component API documentation
  • Implementation guides
  • Accessibility checklists
  • Theming and token documentation

Governance

  • Design Forum integration
  • Proposal templates
  • FOG-based evaluation (Facts, Opinions, Guesses)
  • Quarterly audits

Measurement Framework

I defined and implemented a comprehensive metrics framework measuring both internal and external impact:

Internal Impact Metrics

  • Design velocity
  • Engineering velocity
  • Component adoption rate
  • Accessibility compliance
  • UX designer satisfaction

External Impact Metrics

  • Experience consistency
  • Visual cohesion
  • Reduced user confusion
  • Improved product onboarding

We tracked success week-over-week using Jira, Confluence analytics, and UX sentiment surveys.
